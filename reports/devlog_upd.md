## Установка

Для работы с движком нам понадобился Python версии 3.7 или выше. Рекомендуется использовать самую свежую версию. Примечание: Python 2 не поддерживается.

[Скачать Python можно здесь.](https://www.python.org/downloads/)

Также потребуется последняя версия библиотеки TCOD.

[Инструкции по установке TCOD находятся здесь.](https://python-tcod.readthedocs.io/en/latest/installation.html)

Хотя установить TCOD и сделать игру можно и без этого (как и поступили мы), я настоятельно рекомендую использовать виртуальное окружение. [Документация по его настройке доступна здесь.](https://docs.python.org/3/library/venv.html)

Кроме того, если вы решите работать с виртуальным окружением, стоит потратить немного времени и создать файл `requirements.txt`. Это позволит вам отслеживать зависимости проекта, если вы добавите новые в будущем, и упростит их установку (например, при клонировании из удалённого git-репозитория).

Файл `requirements.txt` можно разместить в той же папке, где вы собираетесь вести проект. Создайте файл `requirements.txt` и поместите в него следующие строки:

```
tcod>=11.13
numpy>=1.18
```

После этого, активировав виртуальное окружение, выполните команду:

`pip install -r requirements.txt`

Так вы установите библиотеку TCOD вместе с её зависимостью numpy.

В зависимости от вашего компьютера может также потребоваться установка SDL2. Инструкции по её установке для разных операционных систем уточняйте отдельно. Например, в Ubuntu это можно сделать командой:

`sudo apt-get install libsdl2-dev`

## Процедурная генерация (24.03.2026)

Теперь, когда в наличии у нас были все необходимые ассеты, было решено не тестировать игру на одной заранее заготовленной комнате, а сразу создать процедурную генерацию подземелья.

За создание комнат отвечает функция `RectangularRoom`:
```
class RectangularRoom:
    def __init__(self, x: int, y: int, width: int, height: int):
        self.x1 = x
        self.y1 = y
        self.x2 = x + width
        self.y2 = y + height

    @property
    def inner(self) -> Tuple[slice, slice]:
        """Return the inner area of this room as a 2D array index."""
        return slice(self.x1 + 1, self.x2), slice(self.y1 + 1, self.y2)
```

За создание проходов же - метод `tunnel_between`:

```
def tunnel_between(
    start: Tuple[int, int], end: Tuple[int, int]
) -> Iterator[Tuple[int, int]]:
    """Return an L-shaped tunnel between these two points."""
    x1, y1 = start
    x2, y2 = end
    if random.random() < 0.5:  # 50% chance.
        # Move horizontally, then vertically.
        corner_x, corner_y = x2, y1
    else:
        # Move vertically, then horizontally.
        corner_x, corner_y = x1, y2

    # Generate the coordinates for this tunnel.
    for x, y in tcod.los.bresenham((x1, y1), (corner_x, corner_y)).tolist():
        yield x, y
    for x, y in tcod.los.bresenham((corner_x, corner_y), (x2, y2)).tolist():
        yield x, y
```

## Враги и их ИИ (05.04.2026)

Теперь, когда работающее (и каждый раз новое) подземелье у нас имеется, мы принялись за создание того, что делает roguelike игры таковыми - противников.

Всего в игре у нас их три типа - Орк (слабый враг), Тролль (средний) и Демон (редкий, но сильный). В каждую комнату нужно добавить от 0 до 2 таких врагов, и её наполнение определяется генератором случайных чисел.

Теперь, когда враги генерируются, нужно было заставить их двигаться, преследовать игрока и атаковать. Поскольку наша игра пошаговая, это означает, что у каждого врага, как и у игрока, будет свой ход, и мы будем чередовать их по принципу игрок->все враги->игрок и т.д., и то только, пока враги есть на экране.

Перемещение было сделать несложно. Как только враг оказывается в поле зрения игрока, он начинает преследовать нашего персонажа, двигаясь в его сторону. Атаку было решено сделать ещё проще - "двигаясь" в клетку с врагом, теперь каждая из сторон может наносить урон. Размер урона у врагов зависит от типа, у игрока - от экипировки (которую мы добавим позже).

Пока что было решено добавить одноразовые предметы в генератор подземелья (в маленьком размере) и в механизм создания лута (с некоторой вероятностью каждый враг "роняет" предмет). Нажав на кнопку G вплотную с лежащим предметом, можно подобрать его в инвентарь; нажав на кнопку I, можно инвентарь открыть и предмет использовать.

Первым предметом (и самым простым) стало лечащее зелье, что позволяет восполнить часть здоровья. Затем были созданы три свитка: Молния (очень мощно бьёт по одному ближайшему врагу, даже сквозь стены), Конфуз (заставляет выбранного игроком врага ходить в случайных направлениях на несколько ходов) и Огненный шар (атакует выделяемую игроком область). Если один из этих свитков (кроме Молнии) не задевает ни одного врага, он не тратится.

Также мы настроили частоту появления, чтобы свитки появлялись реже.

## Сохранение и загрузка (09.04.2026)

Сохранение и загрузка почти на 100% обязательны для каждого рогалика, но могут стать той ещё занозой, если взяться за них слишком поздно.

Был создан новый класс, `EventHandler`, который отвечает за различные ошибки и игровые состояния. Теперь мы можем создавать сохранения автоматически при выходе из игры, создать состояние конца игры (и не делать автосохранение в этом случае).

Для того, чтобы эта работа стала полезной, мы создали главное меню, в котором можно загрузить сохранение или же начать новую игру.

Теперь пришло время создать сам файл сохранения. Мы используем формат .sav, чтобы хранить файл `savegame.sav`. В нём хранятся координаты игрока, всех монстров подземелья, комнат подземелья и, когда эта функция будет добавлена, номер этажа.

## Новые этажи, усложнение игры и экипировка (15.05.2026)

Классический roguelike продолжается до бесконечности, постепенно становясь сложнее до тех пор, пока продолжать будет уже невозможно (или, в некоторых случаях, до заранее определенного потолка сложности). Как следствие, было необходимо добавить новые подземелья (здесь именуемые "этажами" одного и того же подземелья), систему постепенного усложнения и улучшенной экипировки.

Для начала автогенератор подземелья был изменён. Теперь вместо фиксированного количества комнат (и монстров на комнату) эти значения можно менять в процессе игры. Также на каждом этаже подземелья в одной из комнат (кроме начальной) будет появляться особая клетка спуска, что позволяет пройти на "этаж" ниже (перейти на следующий уровень). Далее были добавлены "веса" (weights), которые отвечают за изменение параметров генерации подземелья (например, увеличивать количество врагов и уменьшать предметы).

Поскольку генерация конкретного подземелья уже была запрограммирована нами, теперь всё, что нам оставалось сделать - написать код, что создаст новое подземелье каждый раз, когда мы используем "спуск", изменяя "веса" подходящим образом.

И, наконец, требовалось создать экипировку для прогрессии силы игрока. Для этого были созданы два класса вещей - Оружие (усиливает урон) и Броня (снижает вражеский урон). Пока что процедурное усиление вещей на последующих этажах не добавлено - каждый предмет усиливает игрока на конкретное, заранее заданное значение. При нахождении экипировки в подземелье её можно подобрать клавишей G, после чего в меню экипировки E можно менять, какой из предметов используется. Разумеется, надет на героя может быть только один комплект доспехов и только одно оружие, но в инвентаре E храниться их может сколько угодно.

# Итоги разработки

В результате разработки нашей игры были реализованы все заявленные нами механики. Игрок может исследовать процедурно генерируемые подземелья, что становятся сложнее по мере их прохождения. В этом ему помогут два вида предметов - одноразовые и экипируемые, и они появляются в случайных комнатах подземелья или падают с врагов. Сами враги имеют три типа, разной категории сложности и редкости. Игру можно сохранять и загружать.

В будущем, при необходимости, проект можно расширить. Неиспользуемые предметы экипировки в инвентаре можно превратить из бесполезного мусора в ценный материал для продажи - для чего нужно будет создать особые комнаты магазинов (и систему денег). Экипировку можно будет улучшать в особых комнатах кузни за деньги, накопленные от продаж вещей. Также можно расширить ассортимент одноразовых предметов и экипировки, сделать прогрессирующее улучшение предметов, падающих с врагов более высокой сложности (к примеру, с Демонов), чтобы награждать игроков за поиск опасностей вместо бегства. Быть может, будет полезным создать макропрогрессию в виде улучшений базовых характеристик персонажа за достижение глубоких этажей (например, за каждые 5 этажей в глубину).
